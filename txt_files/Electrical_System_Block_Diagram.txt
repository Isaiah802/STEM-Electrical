═══════════════════════════════════════════════════════════════════════════════════════════
   POWER FLOW SUMMARY
═══════════════════════════════════════════════════════════════════════════════════════════

14.8V RAIL (High Power):
  Source:  LiPo Battery
  Consumers:
    - ESC Input (Motor Power)           10-18A typical, 30A max
    - External UBEC Input               ~1.5A (for 5V conversion)
  Total Current: 11.5-19.5A typical

5V RAIL 1 (Flight-Critical): 
  Source: ESC BEC
  Capacity: 3A
  Consumers:
    - Flight Controller                 250mA
    - GPS                                60mA
    - Receiver                           90mA
    - Arduino Nano + Sensors            263mA
  Total Current: 663mA (22% of capacity)
  Margin: 2337mA (78%) ✓ EXCELLENT

5V RAIL 2 (Servo Power):
  Source: External UBEC  
  Capacity: 6A
  Consumers:
    - 4× Servos (idle)                   40mA
    - 4× Servos (moving)                720mA typical, 1200mA peak
  Total Current: 720mA typical, 1200mA peak
  Margin: 4800mA typical (80%) ✓ EXCELLENT

GROUND:
  ⚠ CRITICAL: ALL grounds must be connected together
    - LiPo negative
    - ESC ground
    - External UBEC ground
    - Flight Controller ground
    - Arduino ground
    - All sensor grounds
  
  Use star grounding topology from battery negative terminal.


═══════════════════════════════════════════════════════════════════════════════════════════
   KEY DESIGN PRINCIPLES
═══════════════════════════════════════════════════════════════════════════════════════════

1. ISOLATION
   Flight-critical systems (FC, GPS, RX) are powered separately from 
   high-current servo loads. This prevents voltage sag/brownout on the FC.

2. REDUNDANCY
   If servo BEC fails, flight systems remain powered. Aircraft can still
   be controlled (though without servo movement - glide to landing).

3. COMMON GROUND
   All subsystems share a common ground reference to prevent ground loops
   and ensure clean signal integrity.

4. NOISE ISOLATION
   Motor power wires are physically separated from:
   - GPS antenna
   - Receiver antenna  
   - Analog sensor (MQ135)
   
   Twist motor wires to reduce electromagnetic interference.

5. POWER MARGIN
   Each power rail has >75% margin under typical operation, allowing for:
   - Component tolerance variation
   - Aging/degradation
   - Unexpected current spikes
   - Engineering safety factor

6. FAIL-SAFE DESIGN
   System degradation modes:
   - Servo BEC fails → Glide to landing (no control surfaces)
   - FC BEC fails → Loss of control (catastrophic - prevented by isolation)
   - Arduino fails → Flight continues, no data logging
   - GPS fails → Manual flight mode
   - Receiver fails → Pre-programmed failsafe (circle & land)

═══════════════════════════════════════════════════════════════════════════════════════════
   RECOMMENDED BUILD SEQUENCE
═══════════════════════════════════════════════════════════════════════════════════════════

PHASE 1: Power Distribution (Week 1, Day 1-2)
  1. Solder XT60 Y-harness (Battery → ESC + External BEC)
  2. Verify External BEC 5V output with multimeter
  3. Create power distribution board or harness for 5V rails
  4. Label all power wires clearly (Rail 1 / Rail 2)

PHASE 2: Flight Systems (Week 1, Day 3-4)
  5. Connect ESC to motor (verify rotation direction)
  6. Connect FC to ESC BEC (Rail 1)
  7. Connect GPS to FC
  8. Connect Receiver to FC
  9. Bind receiver to transmitter
 10. Configure FC firmware (iNav or ArduPilot)

PHASE 3: Servo Systems (Week 1, Day 4-5)
 11. Connect all 4 servos to External BEC (Rail 2)
 12. Connect servo signal wires to FC outputs (Ch1-4)
 13. Test servo movement with transmitter
 14. Set servo endpoints and reversing in FC

PHASE 4: Research Payload (Week 2, Day 1-2)
 15. Wire Arduino to Rail 1 (or Rail 2)
 16. Connect BME280 to Arduino (I2C: SDA, SCL)
 17. Connect MQ135 to Arduino (Analog: A0)
 18. Connect MicroSD module to Arduino (SPI)
 19. Upload data logging code
 20. Test data logging on bench

PHASE 5: Integration & Testing (Week 2, Day 3-5)
 21. Measure all system currents with clamp meter
 22. Verify common ground continuity
 23. Check for electrical noise (GPS lock during motor run)
 24. Final wire management (zip ties, heat shrink)
 25. Install in airframe
