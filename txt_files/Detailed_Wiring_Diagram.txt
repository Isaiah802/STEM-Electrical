┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

  Recommended: Hobbywing UBEC 5V 6A or Castle Creations BEC 2.0

  INPUT (Battery Power):
    ┌──────────────────┐
    │   RED    BLACK   │  XT30 or JST connector
    │  ┌───┐  ┌───┐   │  (or bare wires with heat shrink)
    │  │ + │  │ - │   │
    └──┴───┴──┴───┴───┘
       14.8V    GND
     [16-18 AWG]

  OUTPUT (5V Regulated):
    ┌──────────────────┐
    │   RED    BLACK   │  JST/XT30 or bare wires
    │  ┌───┐  ┌───┐   │  
    │  │ + │  │ - │   │
    └──┴───┴──┴───┴───┘
       +5.0V    GND
       @ 6A     [RAIL 2]
     [18-20 AWG]

  WIRING INSTRUCTIONS:
    1. Solder XT30 or XT60 pigtail to INPUT (match battery connector)
    2. Route OUTPUT wires to servo power distribution
    3. Use 18-20 AWG wire for output (6A current capacity)
    4. Add 470µF capacitor across output (reduces voltage ripple)
    5. Heat shrink all connections

  SPECIFICATIONS:
    • Input Voltage: 5.5-26V (2-6S LiPo compatible)
    • Output Voltage: 5.0V ± 0.1V
    • Output Current: 6A continuous, 8A peak (2 sec)
    • Efficiency: ~90%
    • Switching Frequency: ~300kHz (very low noise)
    • Weight: ~15g

  CONNECTION TO SERVO RAIL:
    +5V → Connect to RED wires of all 4 servos (parallel)
    GND → Connect to BROWN wires of all 4 servos + common ground

  MOTOR DIRECTION TEST:
    1. Remove propeller
    2. Power on ESC (motor should NOT spin at idle)
    3. Slowly advance throttle
    4. Motor should spin CLOCKWISE (viewed from front)
    5. If wrong direction: Power off, swap ANY TWO motor wires

  SPECIFICATIONS:
    • KV Rating: 1000 (RPM per volt)
    • Voltage: 3-4S LiPo (11.1-14.8V)
    • Max Current: 18A @ 14.8V
    • Shaft Diameter: 3.17mm (1/8")
    • Weight: 58g
    • Propeller: 1060 or 1160 folding prop

  ⚠ SAFETY:
    • NEVER test motor with propeller indoors
    • Keep fingers/clothing away from moving parts
    • Disconnect battery before prop installation
    • Use Loctite on prop nuts/bolts

        ├─> Motor GND             └─> Servo Power GND       ├─> GPS GND  │          │
        └─> BEC Output GND                                  └─> RX GND   │          │
                                                                          │          │
                                                                          └─> Sensor GND
                                                                          └─> SD GND

⚠ WARNINGS:
  - Do NOT create ground loops (multiple parallel ground paths)
  - Ensure ALL components share the same ground reference
  - Use thick wire (18-20 AWG) for main ground bus
  - Poor grounding causes: erratic behavior, resets, noise, GPS dropouts

═══════════════════════════════════════════════════════════════════════════════════════════
   SECTION 6: TESTING PROCEDURE (STEP-BY-STEP)
═══════════════════════════════════════════════════════════════════════════════════════════

⚠ ALWAYS TEST SUBSYSTEMS BEFORE FULL INTEGRATION

PHASE 1: Component-Level Testing
──────────────────────────────────────────────────────────────────────────────────────────
Step 1.1: Battery Voltage Check
  ☐ Measure battery voltage with multimeter: Should read 14.8V nominal (±1V)
  ☐ Check each cell voltage via balance connector: All cells ~3.7V
  ☐ If any cell <3.5V, charge battery before proceeding

Step 1.2: ESC BEC Output Test
  ☐ Connect battery to ESC (NO MOTOR connected yet)
  ☐ Measure BEC output with multimeter: Should read 5.0V ± 0.1V
  ☐ If voltage incorrect, ESC may be faulty or misconfigured

Step 1.3: External BEC Test
  ☐ Connect battery to External BEC
  ☐ Measure output with multimeter: Should read 5.0V ± 0.1V
  ☐ Load test: Connect 1Ω resistor (5W rated) → Should still read 5.0V

Step 1.4: Servo Movement Test (Individual)
  ☐ Connect ONE servo to External BEC (power only, no signal)
  ☐ Manually move servo arm: Should move smoothly, no binding
  ☐ Repeat for all 4 servos

Step 1.5: Arduino Power Test
  ☐ Connect Arduino to 5V + GND from Rail 1
  ☐ Verify onboard LED lights up
  ☐ Upload simple blink sketch to verify functionality

──────────────────────────────────────────────────────────────────────────────────────────
PHASE 2: Subsystem Integration Testing
──────────────────────────────────────────────────────────────────────────────────────────
Step 2.1: Flight Controller Power-Up
  ☐ Connect FC to ESC BEC (Rail 1)
  ☐ Verify FC status LED (should blink)
  ☐ Connect to FC via USB, access configuration software
  ☐ Verify firmware version and basic settings

Step 2.2: GPS Connection
  ☐ Connect GPS to FC (UART1)
  ☐ Power system, wait 2-5 minutes for GPS lock
  ☐ In FC software, verify GPS shows satellite count (>8 for good lock)
  ☐ If no lock, check antenna orientation and wiring

Step 2.3: Receiver Binding
  ☐ Connect receiver to FC
  ☐ Bind receiver to transmitter (follow ELRS binding procedure)
  ☐ In FC software, verify receiver channels responding (move sticks)
  ☐ Check all 4 channels: Throttle, Aileron, Elevator, Rudder

Step 2.4: ESC Calibration & Motor Test
  ☐ Connect ESC signal wire to FC motor output (M1)
  ☐ Calibrate ESC (follow ESC manual procedure)
  ☐ ⚠ NO PROPELLER: Test motor spin at low throttle
  ☐ Verify motor direction (clockwise viewed from front)
  ☐ If reversed, swap any two motor wires

Step 2.5: Servo Signal Test
  ☐ Connect all 4 servos to FC PWM outputs (S1-S4)
  ☐ Configure servo outputs in FC software (set to "Servo" mode)
  ☐ Move TX sticks, verify servo response:
      - Right stick left/right → Ailerons move (S1 up, S2 down or vice versa)
      - Right stick up/down → Elevator moves
      - Left stick left/right → Rudder moves
  ☐ Adjust servo direction (reverse) in FC if needed
  ☐ Set servo endpoints (typically 1000-2000µs, may limit to prevent binding)

──────────────────────────────────────────────────────────────────────────────────────────
PHASE 3: Research Payload Testing
──────────────────────────────────────────────────────────────────────────────────────────
Step 3.1: BME280 Sensor Test
  ☐ Wire BME280 to Arduino (I2C: SDA → A4, SCL → A5)
  ☐ Upload BME280 test sketch (from Adafruit library examples)
  ☐ Open Serial Monitor, verify readings:
      - Temperature: Reasonable room temp (~20-25°C)
      - Humidity: 30-70% typical indoors
      - Pressure: 980-1020 hPa (varies by altitude)

Step 3.2: MQ135 Sensor Test
  ☐ Wire MQ135 to Arduino (AOUT → A0)
  ☐ Upload sketch to read analog value from A0
  ☐ Readings will be unstable initially (sensor needs warm-up)
  ☐ Verify analog value changes when blowing on sensor

Step 3.3: MicroSD Module Test
  ☐ Wire SD module to Arduino (SPI: MOSI, MISO, SCK, CS)
  ☐ Insert formatted SD card (FAT32, <32GB)
  ☐ Upload SD card test sketch (from SD library examples)
  ☐ Verify file creation and data write to SD card
  ☐ Remove card, check file on computer

Step 3.4: Toggle Switch Test
  ☐ Wire toggle switch (D2 with INPUT_PULLUP)
  ☐ Upload sketch to read D2 state
  ☐ Verify Serial Monitor shows HIGH (open) and LOW (closed)

Step 3.5: Integrated Data Logging Test
  ☐ Upload final data logging sketch (reads sensors, writes to SD)
  ☐ Toggle switch ON, verify data logging starts
  ☐ Let run for 1 minute
  ☐ Toggle switch OFF, verify logging stops
  ☐ Remove SD card, verify CSV file with timestamped data

──────────────────────────────────────────────────────────────────────────────────────────
PHASE 4: Full System Integration
──────────────────────────────────────────────────────────────────────────────────────────
Step 4.1: Current Draw Measurement
  ☐ Use clamp meter or inline current sensor
  ☐ Measure Rail 1 current (ESC BEC output):
      - Idle (no servos moving): ~320-400mA ✓
      - With servo movement: ~1000-1400mA ✓
  ☐ Measure Rail 2 current (External BEC):
      - Idle: ~40mA ✓
      - With servo movement: ~700-1200mA ✓
  ☐ If currents exceed estimates, investigate (possible short/malfunction)

Step 4.2: Voltage Sag Test
  ☐ Connect multimeter to FC 5V input
  ☐ Move all servos simultaneously (full deflection)
  ☐ FC voltage must stay >4.8V (preferably >4.9V)
  ☐ If voltage sags below 4.8V, FC may brown out → Increase BEC capacity

Step 4.3: Electrical Noise Test
  ☐ Power entire system, achieve GPS lock
  ☐ Spin motor at 50% throttle (NO PROP)
  ☐ Monitor GPS satellite count in FC software
  ☐ GPS should NOT lose lock or show reduced satellite count
  ☐ If GPS affected: Twist motor wires, add ferrite bead, increase separation

Step 4.4: Ground Continuity Test
  ☐ Use multimeter in continuity mode (beep test)
  ☐ Verify 0Ω resistance between:
      - Battery negative ↔ ESC ground
      - Battery negative ↔ FC ground
      - Battery negative ↔ Arduino ground
      - Battery negative ↔ Servo ground
  ☐ If no continuity, fix ground connections immediately

Step 4.5: Receiver Range Test
  ☐ Power system, bind TX/RX
  ☐ Walk 50m away with transmitter
  ☐ Verify control inputs still responsive (move servos)
  ☐ Walk to 100m if safe to do so
  ☐ ELRS should achieve >1km range in open field (overkill for this project)

──────────────────────────────────────────────────────────────────────────────────────────
PHASE 5: Pre-Flight Validation
──────────────────────────────────────────────────────────────────────────────────────────
Step 5.1: Control Surface Check
  ☐ With aircraft on bench, verify control surface movement:
      - Aileron stick right → Right aileron down, Left aileron up
      - Aileron stick left → Right aileron up, Left aileron down
      - Elevator stick down → Elevator up (pitch up)
      - Elevator stick up → Elevator down (pitch down)
      - Rudder stick right → Rudder right
      - Rudder stick left → Rudder left
  ☐ If reversed, fix in FC configuration

Step 5.2: Failsafe Test
  ☐ Power aircraft, bind TX/RX
  ☐ Set throttle to mid-stick
  ☐ Turn OFF transmitter
  ☐ Verify failsafe activates (servos should hold position or center)
  ☐ Motor should cut (throttle → 0%)
  ☐ Configure failsafe in FC if behavior incorrect

Step 5.3: Battery Low Voltage Alarm
  ☐ Configure low voltage alarm in FC (set to 14.0V or 3.5V/cell)
  ☐ If FC has buzzer, verify audible alarm
  ☐ Alternatively, configure telemetry alert on TX display

Step 5.4: Data Logging Pre-Flight Check
  ☐ Power Arduino, toggle switch ON
  ☐ Verify logging starts (LED indicator if implemented)
  ☐ Let log for 30 seconds
  ☐ Toggle OFF, remove SD card, check file
  ☐ Plan to power Arduino 24-48 hours before flight (MQ135 warm-up)

Step 5.5: Final Visual Inspection
  ☐ Check all connectors secure (no loose wires)
  ☐ Verify no exposed metal (risk of shorts)
  ☐ Check wire routing (away from control surfaces, no chafing)
  ☐ Verify all components secured (no loose parts in flight)
  ☐ Confirm propeller secure (but REMOVE for ground testing!)


═══════════════════════════════════════════════════════════════════════════════════════════
   SECTION 7: COMMON WIRING MISTAKES & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════════════════

PROBLEM: Flight controller not powering on
  Cause 1: No power to FC 5V/GND pads
    → Check ESC BEC output voltage (should be 5V)
    → Verify FC 5V pad solder joint (may be cold joint)
  Cause 2: FC damaged (rare)
    → Try powering via USB to isolate issue

PROBLEM: Servos jittering or not moving smoothly
  Cause 1: Insufficient power (voltage sag)
    → Verify servo power from Rail 2 (External BEC), not FC
    → Measure voltage at servo power rail under load (must be >4.8V)
  Cause 2: Noisy PWM signal
    → Shorten signal wires, route away from motor/ESC wires
  Cause 3: Mechanical binding
    → Check control surface linkages for friction

PROBLEM: GPS not getting lock (satellite count = 0)
  Cause 1: No power to GPS
    → Verify GPS VCC connected to FC 5V
  Cause 2: Wrong UART port or baud rate
    → Check FC configuration (typically UART1, 9600 or 115200 baud)
  Cause 3: GPS antenna obstructed
    → Mount GPS with clear view of sky, away from carbon fiber
  Cause 4: Indoor testing
    → GPS requires outdoor testing for lock

PROBLEM: Receiver not binding to transmitter
  Cause 1: Wrong binding procedure
    → Follow ELRS binding steps (bind button or WiFi method)
  Cause 2: Receiver not powered
    → Verify receiver VCC connected to 5V, check current draw (~90mA)
  Cause 3: Firmware mismatch
    → Ensure TX and RX have same ELRS firmware version

PROBLEM: Motor not spinning
  Cause 1: ESC not armed
    → Throttle must be at zero during power-up to arm ESC
  Cause 2: No signal to ESC
    → Verify ESC signal wire connected to FC motor output (M1)
  Cause 3: ESC not calibrated
    → Run ESC calibration procedure (throttle range learning)

PROBLEM: Arduino sensors not reading correctly
  Cause 1: Wrong I2C address (BME280)
    → Run I2C scanner sketch, check address (0x76 or 0x77)
  Cause 2: MQ135 unstable readings
    → Sensor needs 24-48 hour warm-up for stability
  Cause 3: SD card not detected
    → Verify card formatted FAT32, check SPI wiring
    → Try different SD card (some cards incompatible)

PROBLEM: System resets randomly during operation
  Cause 1: Voltage brownout (power supply insufficient)
    → Check battery voltage (must be >13.2V)
    → Measure voltage at FC under load (must be >4.8V)
  Cause 2: Poor ground connection
    → Verify common ground continuity across all components
  Cause 3: Electrical noise from motor/ESC
    → Add capacitor across motor terminals, twist motor wires
